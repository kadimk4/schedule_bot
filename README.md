# Бот расписания для групп

Умный телеграм-бот для отслеживания расписания занятий с поддержкой нескольких групп и мощной системой управления ролями (Админ, Староста, Студент).

## 🚀 Быстрый старт (Docker)

1. Создайте `.env` файл (используйте `BOT_TOKEN` и `ADMIN_ID`).
2. Запустите бота через Makefile:
```bash
make build
make up
```

## 🛠 Функционал и Команды

### 👑 Панель Администратора (`/admin`)
Интерактивное меню для полного контроля над ботом:
*   **Список пользователей**: Просмотр всех зарегистрированных людей, их ролей и групп.
*   **Управление пользователем**: При нажатии на имя можно:
    *   Сменить группу (выбор из списка существующих).
    *   Назначить старостой или студентом.
    *   Удалить пользователя из базы данных.
*   **Управление группами**: Просмотр списка групп и их удаление.
*   **Команды**:
    *   `/add_group <название>` — Быстрое создание группы.
    *   `/set_group @username <группа>` — Добавить пользователя в группу как студента.
    *   `/set_starosta @username <группа>` — Назначить старосту для группы.

### 🎓 Панель Старосты (`/starosta`)
Доступна пользователям с ролью `starosta` (и админу для тестов):
*   **🔗 Ссылка для вступления**: Генерирует уникальную инвайт-ссылку. Студенты, перешедшие по ней, автоматически привязываются к группе старосты.
*   **👥 Список группы**: Позволяет видеть всех студентов своей группы и исключать их при необходимости.

### 📖 Функции для студентов
*   **🗓 на сегодня / на завтра** — Расписание на конкретный день.
*   **▶️ Сейчас / ⏭ Следующая** — Информация о текущей и предстоящей паре (требуется формат `08:30 - 10:00 | текст` в JSON).
*   **📅 Выбрать день** — Инлайн-меню для выбора любого дня недели или просмотра всей недели сразу.

## 📁 Настройка расписания
Расписание хранится в `data/schedule.json`. Формат поддерживает несколько групп:
```json
{
  "05-507": {
    "Monday": ["08:30 - 10:00 | 🕒 Лекция: Алгебра", "..."],
    "Tuesday": ["..."]
  },
  "05-508": { ... }
}
```

## 📝 Управление проектом
Используйте `Makefile` для удобства:
*   `make up` — Запуск в фоне.
*   `make logs` — Просмотр логов (кто зашел, какие команды использовал).
*   `make restart` — Перезапуск после изменений в коде.
*   `make shell` — Вход внутрь контейнера.
