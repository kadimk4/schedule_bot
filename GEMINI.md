# Состояние проекта NarutoMentality (Бот расписания)

Этот файл содержит контекст разработки для Gemini CLI, чтобы обеспечить преемственность сессий.

## 🏗 Архитектура
Проект разбит на модули для удобства масштабирования:
- `bot.py` — Точка входа, инициализация `Application` и регистрация хендлеров.
- `handlers/` — Логика команд:
    - `admin.py` — Интерактивная панель `/admin`, управление ролями и группами.
    - `starosta.py` — Интерактивная панель `/starosta`, инвайт-ссылки и список группы.
    - `schedule.py` — Обработка запросов расписания (сегодня/завтра/выбор дня).
    - `news.py` — Система рассылки новостей (Admin -> Все, Starosta -> Группа).
- `utils/` — Инструментарий:
    - `database.py` — Асинхронная работа с SQLite (`aiosqlite`). Таблицы `users` и `groups`.
    - `scheduler.py` — Логика парсинга `schedule.json` и вычисления текущих/следующих пар.
    - `config.py` — Конфигурация через `pydantic-settings`.

## 🗄 База данных (SQLite)
Таблицы:
- `users`: `tg_id`, `username`, `role` (admin/starosta/student), `group_id`.
- `groups`: `id`, `name`.

## 🐋 Docker & Окружение
- Используется `docker-compose` с `volumes` для сохранения базы данных (`data/database.db`) и кода.
- Пакетный менеджер: `uv`.
- Управление через `Makefile`.

## 🛠 Важные заметки для будущих сессий
1. **Расписание**: В `schedule.json` используется формат `08:30 - 10:00 | Текст` для корректной работы функций времени.
2. **Админ**: Твой ID (`1032588053`) прописан в `.env`. Бот автоматически назначает тебя админом при инициализации БД.
3. **Безопасность**: Все callback-запросы панелей (`adm_`, `st_`) проверяются на наличие соответствующих ролей в БД.
4. **Новости**: Рассылка реализована через `ConversationHandler` в `handlers/news.py`.
